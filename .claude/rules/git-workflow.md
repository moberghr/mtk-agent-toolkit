# Git & Workflow (S4)

> Generated by init on 2026-04-11. Cite rules as S4.N in reviews.

## Branching

- **S4.1** Use hierarchical branch names with `/`: `feat/skill-name`, `fix/hook-bug`, `docs/readme-update`.
- **S4.2** Main branch is `main`. All work branches off and merges back to `main`.

## Commits

- **S4.3** Commit messages use conventional format: `type: description`. Types: `feat`, `fix`, `refactor`, `docs`, `chore`.
- **S4.4** Keep commits focused — one logical change per commit. Don't bundle unrelated changes.

## Versioning

- **S4.5** Follow semver: major (breaking changes to skill contracts), minor (new skills), patch (fixes, tweaks).
- **S4.6** Bump version in both `manifest.json` and `plugin.json` in the same commit.
- **S4.7** Update `manifest.updated` date when bumping version.

## Release Checklist

- **S4.8** Before release: `bash scripts/validate-toolkit.sh` must pass.
- **S4.9** Before release: README must reflect any new skills.
- **S4.10** Before release: AGENTS.md must route to any new skills.

## Critical Rules (Always Apply)

### C1 ‚Äî Actor & concurrency discipline
- **C1.1** One actor = one mailbox = one thread. Never `lock` inside an actor's `Receive` ‚Äî messages are already serialized per actor.
- **C1.2** Actor state is mutable inside the actor, **invisible** outside. No public mutable fields. No property getters that return actor state from outside.
- **C1.3** Use `Ask<T>` only for request/response with a timeout. For fire-and-forget, use `Tell`.
- **C1.4** Supervision strategy is an explicit decision ‚Äî `OneForOneStrategy` by default; document when `AllForOne` is justified.
- **C1.5** Distributed work uses sharding or cluster singletons, not manual role routing. Entity IDs must be stable.

### C2 ‚Äî Data & persistence
- **C2.1** Money types are `decimal`. Reject `double`/`float` for monetary fields (the finance linter pack catches this).
- **C2.2** Every state mutation that touches audited entities must write to the audit trail in the same transaction.
- **C2.3** EF Core reads default to `AsNoTracking()`. Tracking is opt-in and justified in comments or commit messages.
- **C2.4** No `Include()` chains followed by `Select(new ‚Ä¶)` ‚Äî the projection makes the include redundant and fetches unused columns.
- **C2.5** Migrations run in a dedicated migration service, not on app startup.

### C3 ‚Äî Security & boundaries
- **C3.1** Every public API endpoint has `[Authorize(Policy = ‚Ä¶)]`. No "internal endpoint" exemptions.
- **C3.2** Raw SQL uses `FromSqlInterpolated` (parameterized). Never `FromSqlRaw` with string interpolation.
- **C3.3** Secrets read from configuration/`IOptions<T>` ‚Äî never string literals in code.
- **C3.4** No PII in structured logs. Use redaction helpers for email, phone, account numbers.

### C4 ‚Äî Testing
- **C4.1** Every new handler has a unit test for the happy path and every error path that can be triggered.
- **C4.2** Akka tests use `Akka.Hosting.TestKit` TestProbes for interaction verification ‚Äî no mocking the actor system.
- **C4.3** Integration tests use TestContainers against a real PostgreSQL ‚Äî never in-memory EF when relational behavior matters.
- **C4.4** No `[Skip]` on tests without a linked issue. The slopwatch linter flags this.

### C5 ‚Äî MediatR slice discipline
- **C5.1** One feature = one vertical slice in `Features/<Feature>/`. Handler + request + response + validator co-located.
- **C5.2** Controllers are thin. Business logic lives in handlers. (Compliance reviewer flags logic-in-controller.)
- **C5.3** Validators run before the handler ‚Äî rely on FluentValidation pipeline behavior, not inline checks.

# Slopwatch — LLM reward hacking patterns.
# Detects code that games quality metrics: disabled tests, suppressed warnings,
# empty implementations, and assertions that always pass.
# Tab-separated: RULE_ID<TAB>SEVERITY<TAB>ERE_REGEX<TAB>RATIONALE<TAB>SUGGESTED_FIX
SLOP-SKIP-TEST	warning	\[(Skip|Ignore|Fact\(Skip)|Theory\(Skip)|xfail|pytest\.mark\.skip|\.skip\(|xit\(|xdescribe\(	Test disabled — may be masking a failure	Re-enable the test or document why it must stay disabled
SLOP-PRAGMA-NORESTORE	warning	#pragma warning disable(?!.*#pragma warning restore)	Warning suppressed without restore — hides issues for entire file	Add matching #pragma warning restore after the affected block
SLOP-TODO-IMPL	warning	(TODO|FIXME|HACK|XXX).*(?i)(implement|finish|complete|add logic|fill in)	Placeholder replacing real implementation	Implement the actual logic before marking task complete
SLOP-EMPTY-CATCH	warning	catch\s*(\([^)]*\))?\s*\{\s*\}	Empty catch block silently swallows errors	Handle the exception, log it, or add a comment explaining why it is intentionally ignored
SLOP-ASSERT-TRUE	warning	Assert\.(True|IsTrue)\(\s*true\s*\)|assert\s+True|expect\(true\)\.toBe\(true\)	Assertion that always passes — tests nothing	Replace with a meaningful assertion on actual behavior
SLOP-NOOP-TEST	warning	\[(Fact|Test|Theory)\]\s*\n\s*public\s+(async\s+)?[A-Za-z<>]+\s+\w+\([^)]*\)\s*\{\s*(return;?)?\s*\}	Test method with empty body — counts as passing but tests nothing	Add assertions that verify actual behavior
